     753              : // Comparison operator implementation. The library supplied operators
     754              : // enables the user to provide fully interoperable constant/mutable
     755              : // iterator types. I.e. the library provides all operators
     756              : // for all mutable/constant iterator combinations.
     757              : //
     758              : // Note though that this kind of interoperability for constant/mutable
     759              : // iterators is not required by the standard for container iterators.
     760              : // All the standard asks for is a conversion mutable -> constant.
     761              : // Most standard library implementations nowadays provide fully interoperable
     762              : // iterator implementations, but there are still heavily used implementations
     763              : // that do not provide them. (Actually it's even worse, they do not provide
     764              : // them for only a few iterators.)

     766              : // ?? Maybe a BOOST_ITERATOR_NO_FULL_INTEROPERABILITY macro should
     767              : //    enable the user to turn off mixed type operators
     768              : //
     769              : // The library takes care to provide only the right operator overloads.
     770              : // I.e.
     771              : //
     772              : // bool operator==(Iterator,      Iterator);
     773              : // bool operator==(ConstIterator, Iterator);
     774              : // bool operator==(Iterator,      ConstIterator);

     776              : //
     777              : //   ...

     780              : // by current compiler releases. The library is designed to degrade gracefully
